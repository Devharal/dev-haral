## What Is Docker?

Docker is a **containerization platform**.

A **container** is a lightweight, isolated runtime environment that:

- Runs your application
- Includes all dependencies
- Behaves the same everywhere

---

## Containers vs Virtual Machines



| Virtual Machines | Containers |
|------------------|------------|
| Heavy | Lightweight |
| Full OS per VM | Share host OS |
| Slow boot | Instant startup |
| High resource usage | Efficient |

## Core Docker Concepts

### Docker Image

An **image** is a blueprint:

- Base OS
- Runtime (Node, Python, Java, etc.)
- Dependencies
- Application code

Images are **immutable**.

---

### Docker Container

A **container** is a running instance of an image.

> Image = Class  
> Container = Object

---

### Dockerfile

A **Dockerfile** defines how to build an image.

Think of it as a recipe.

## Creating Your First Dockerfile (Node.js)

**Project structure**

```text
app/
 â”œâ”€ index.js
 â”œâ”€ package.json
 â””â”€ Dockerfile
```

**index.js**

```js
const express = require("express");
const app = express();

app.get("/", (req, res) => res.send("Hello from Docker"));

app.listen(3000, () => console.log("Running on 3000"));
```

**Dockerfile**

```text
FROM node:18-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .

EXPOSE 3000
CMD ["node", "index.js"]
```

## Docker Compose (Multiple Services)

```yaml
version: "3.8"
services:
  app:
    build: .
    ports:
      - "3000:3000"
    depends_on:
      - redis

  redis:
    image: redis:7
```

```bash
docker-compose up
```
